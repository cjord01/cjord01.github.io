Best Practices: Classes vs. IDs

I chose to focus my blog entry on best practices associated with using Classes vs. IDs because I felt like this was the area I was the most clueless. After a little research, I'm now much less clueless (about classes and IDs). Let's start with a few definitions:

  • Classes and IDs are used in "CSS". CSS is a styling language used to create the look and formatting of webpages (mostly used in conjunction with HTML).
  • Classes and IDs are "Selectors". Selectors are used to select the elements of the HTML file that you want to style. Selector syntax looks like this:
  • Classes are identified by a '.'
  • IDs are identified by a '#'

Classes vs IDs

Both Classes and IDs are used to define a special case for an element. The main difference is that IDs only apply to one element of a webpage while lots of elements can have the same Class. For example there is only one footer in a webpage but there are often many paragraphs. So... the footer would be an ID and the paragraphs would be a Class.

Since a class can be used many times and an ID can only be used once, you should use classes for items that will be used multiple times.

Here's a way to help you remember!

  • An "ID" (like a driver's license) is unique to one person.
  • When you take a "class" there are multiple students enrolled.

The rule of thumb is to ask yourself if you think you might need to use the selector for another element on your site. If the answer is yes then go with a class.
